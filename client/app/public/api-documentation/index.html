    Introduction

    DocumentME contains restful API endpoints that allow users to create, read, update, and delete documents. It also has dedicated restful endpoints for the admin which allows deleting a users, creating, updated and deleting roles. These endpoints are only accessed when the user is authenticated

    Development

    DocumentME is a NodeJs application. Sequelize is used as the ORM with Postgresql.

    Installation

    Follow the steps below to setup a local development environment. First ensure you have Postgresql installed, and a version of Node.js greater than or equal to v6.8.0.

    • Clone the repository from a terminal with https://github.com/andela-johia/Document-management-system.git.
    • Move into the project directory with cd Document-management-system
    • Create a .env file and add your database URL as the key name DATABASE URL.
    • Install all project dependencies with npm install
    • Run npm start to start the server.

    API Summary

    Users

    EndPoint Functionality
    POST /users/login signs in a user
    POST /users/logout signs out a user
    POST /users/ Creates a new user
    GET /users/ Gets all users (available only to the Admin)
    GET /users/:id Finds a user by the id specified
    PUT /users/:id Updates a user's details by the id specified (available only to the user)
    DELETE /users/:id Deletes a user by the id specified
    GET /users/?limit={integer}&offset={integer} Get users by limit and/or offset for pagination
    GET /search/users/?q=query Get all users whose name matches the search query (available only to the Admin)

    Documents

    EndPoint Functionality
    POST /documents/ Creates a new document.
    GET /documents/ Gets all documents.
    GET /documents/:id Finds a document by id specified.
    GET /users/:id/documents Gets all documents for a user by the id specified
    PUT /documents/:id Updates a document's details. (available only to the owner)
    DELETE /documents/:id Delete document. (available only to the owner)
    GET /documents/?limit={integer}&offset={integer} Get documents by limit and/or offset for pagination
    GET /search/documents/?q=query Get all documents with title containing the search query

    Roles (available only to the Admin)

    EndPoint Functionality
    GET /roles/ Get all Roles.
    POST /roles/ Create a Role.
    PUT /roles/:id update a Role by id.
    DELETE /roles/:id Delete a Role by id.

    Errors

    The DocumentME API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- This occurs when the request made was a bad request.
    401 Unauthorized -- Your token has not been verified or invalid token.
    403 Forbidden -- The users and roles are hidden for administrators only.
    404 Not Found -- Users or Documents not found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
